# Workflow Testing Checklist

Use this checklist to test the CRM workflow and document possible improvements.

## Applicant Form Testing

- [ ] Add a new applicant with all required fields.
- [ ] Try to submit the form with missing required fields.
- [ ] Edit an existing applicant.
- [ ] Delete an applicant record.
- [ ] Mark all documents as received and confirm the applicant becomes complete.
- [ ] Uncheck one required document and confirm the applicant appears in the incomplete report.

## Search and Filter Testing

- [ ] Search by applicant name.
- [ ] Search by email.
- [ ] Search by program name.
- [ ] Filter by program.
- [ ] Filter by status.
- [ ] Combine search, program filter, and status filter.

## Dashboard Testing

- [ ] Confirm total applicant count updates after adding a record.
- [ ] Confirm incomplete count updates after changing documents.
- [ ] Confirm under-review count updates after changing application status.
- [ ] Confirm admitted count updates after changing application status.
- [ ] Confirm operational alerts update correctly.

## Report Testing

- [ ] Verify applications by program counts.
- [ ] Verify applications by status counts.
- [ ] Verify incomplete applications list.
- [ ] Export applicants CSV.
- [ ] Export summary CSV.
- [ ] Open exported CSV file and check field formatting.

## Help Desk Testing

- [ ] Add a new ticket.
- [ ] Edit ticket priority.
- [ ] Change ticket status to resolved.
- [ ] Delete a ticket.
- [ ] Confirm high-priority unresolved tickets appear in dashboard alerts.

## Usability Testing

- [ ] Check layout on desktop screen.
- [ ] Check layout on smaller/mobile screen.
- [ ] Confirm button labels are understandable.
- [ ] Confirm users can identify next steps easily.
- [ ] Identify any confusing workflow or missing instruction.
